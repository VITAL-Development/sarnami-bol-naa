## adj-sojha finding

Plain "सोझा" reliably mispronounces झ (jha) as ध (dha) — confirmed via
ASR round-trip (facebook/mms-1b-all, hin adapter, run inside the
`rarelang-server-tts-sidecar-1` container's cached HF weights): 0/5 correct
across 5 stochastic samples, and wrong under `--noise-scale 0
--noise-w-scale 0` (deterministic) too.

A syllable break fixes it, but **only the space-separated form** —
"सो झा", not the hyphenated "सो-झा" originally reported. In this session's
testing, hyphenated "सो-झा" was still wrong under the deterministic
(`--noise-scale 0`) setting and only 3/5 under Piper's default stochastic
noise — no better than random. Space-separated "सो झा" was correct under
the deterministic setting and 3/5+ under stochastic sampling. `piper`
doesn't add an audible pause for either form at these settings (confirmed
by ear on the generated wavs).

Piper's default noise-scale makes single-sample ASR checks unreliable —
part 2's automated ASR-check-and-retry loop needs to account for this
(multiple samples per candidate, or a fixed low/zero noise-scale for
reproducibility) rather than trusting one pass.
